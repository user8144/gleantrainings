Lesson 2

When to verify, deprecate, or do neither

Use the lightest effective intervention. Not every noisy search result should trigger a Glean action, and not every important page deserves verification.

Decision shortcut
Ask two questions first:
  1. Is the underlying content actually right?
  2. What is the smallest change that improves trust or reduces confusion?

Decision matrix

SituationBest default actionWhy
The resource is current, accurate, and should carry stronger trust.VerifyYou want to elevate confidence in a source that is already right.
The resource is old, replaced, misleading, or no longer safe to rely on.DeprecateYou want to suppress stale guidance and reduce confusion.
The resource is current but not especially important, or the problem is mostly weak query intent.Do neitherNot every usable page needs stronger trust treatment.
The source is correct, and a high-intent search should consistently route to it.Verify + Pin when appropriateVerification strengthens trust; Pin adds predictable query routing.
The source is wrong or incomplete but business-critical.Fix content firstNever verify bad content just because it matters.

When to verify

  • The page is a true source of truth.
  • The content is current and accurate today.
  • The audience depends on it to make decisions or complete work.
  • A real owner exists and can keep it fresh.

When to deprecate

  • A newer source has replaced it.
  • The content creates repeated confusion.
  • The guidance is still surfacing but should no longer direct behavior.
  • Leaving it active creates trust debt.

When to do neither

  • The page is fine but low-risk and low-traffic.
  • The search problem is really a naming or structure problem upstream.
  • You do not yet know who owns the page.
  • You have not checked whether the content is current.

Where Pin fits

Pin is not trust.

Pin says “for this search, route here.” Verification says “this source is accurate and current.” Use both only when both statements are true.

Avoid these mistakes

  • Verifying something because it is important, even though it is stale.
  • Deprecating content without making sure a good replacement exists.
  • Treating every discoverability issue like a ranking issue.
  • Creating maintenance work no one owns.
