Lesson 5

Use both actions as part of a healthier knowledge layer

A healthy Glean layer is not created by piling on objects. It is created by making current sources easier to trust and stale sources harder to rely on.

End state
What good looks like

Employees are more likely to find the right source first, less likely to get misled by stale content, and more likely to trust the experience enough to self-serve.

A simple operating pattern

1. Find the confusing surface area

Look for the stale guide, duplicate process page, or important source that still lacks trust signals.

2. Fix the source state first

Update, merge, retire, or confirm ownership before adding a Glean action.

3. Apply the right lever

Verify to elevate trust. Deprecate to suppress stale reliance. Add Pin only if query routing also needs control.

4. Re-test and maintain

Search again, sanity-check the user experience, and put the review burden somewhere real.

Quick checklist before you act

  • Do I know who owns this source?
  • Is the content actually current?
  • Is the problem trust, stale competition, or both?
  • Is there a cleaner upstream content fix first?
  • Who will maintain this signal later?

Try this next

  1. Pick one topic area with visible confusion.
  2. Identify one source that deserves stronger trust.
  3. Identify one stale source that should stop competing.
  4. Apply the lightest effective treatment.
  5. Re-run the likely search and see if the experience is cleaner.

One-line takeaway

Verify when you want to say “trust this.” Deprecate when you need to say “do not rely on this.” Use both to keep the knowledge layer current, clean, and easier to navigate.
