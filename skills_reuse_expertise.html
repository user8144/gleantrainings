Concept

A reusable package for a task you do the same way every time

Skills make tasks you ask Glean to do repeatable and help it give more consistent outputs.

What it is

A reusable package of instructions, templates, and tools for a specific task — the structure, standards, and decision logic bundled together.

☵

Like a saved recipe. Once someone writes down how we make the dish well, anyone can cook it the same way.

When to use one

When you'd otherwise rewrite the same structure, standards, or decision logic over and over. A true one-off request usually doesn't need a Skill.

✓

At Cribl, a good Skill captures how we already do a task well — so more people can execute it consistently.

⇆

Agents vs. Skills. A Skill applies packaged instructions to a prompt; an Agent independently executes a multi-step workflow on your behalf. Agents can be given Skills when you want them to follow that prompt.

The shortcut

Type / to skip retyping the same prompt

Two ways a Skill gets applied — and one of them you control directly.

Invoke it yourself

You call it by name

Type / in a Glean chat to pull up your Skills and invoke one directly — instantly applying its structure, tone, and standards without re-explaining them.

Auto-routing

Glean picks it for you

Glean can route to a relevant Skill when your query matches its description — most reliable in thinking mode. Handy, but you don't have to rely on it.

◎

The real time-save: write your formatting rules, sections, and tone once as a Skill, then call it with / — instead of typing them out every time.

01A second example

Turn a resolved Slack thread into a KB article

Scenario: a solved #cribl-support thread needs to become a knowledge base article. Same prompt, two very different outcomes.

Without a Skill
"Write a KB article from this thread."
Inconsistent structure, missing repro steps, no labels applied. Quality depends on whoever happens to be writing it that day.
With a Skill
"Write a KB article from this thread."
Standard sections, consistent tone, and correctly applied labels — every time, no matter who runs it.
◎

Multiply this across everyone who drafts KB articles — and consistency stops depending on who's writing.

02Routing

Auto-routing, and how to turn it on

Auto-routing — when a Skill activates on its own based on keywords — isn't as reliable as calling the Skill yourself with /. But you can enable it so it does its best for you and others. In your Settings, select your Skill, press Share, and turn on Auto-enable; then use the Skill's instructions to define the verbiage it should respond to.

03Trust & access

Skills don't bypass permissions

🔒

Same permission model as always. A Skill only accesses data you already have permission to view.

✗

Never a workaround. Enabling or using a Skill doesn't grant new access to anything you couldn't already see.
